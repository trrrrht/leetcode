**Problem**:

Given an array of integers, return **indices** of the two numbers such that they add up to a specific target.

You may assume that each input would have **exactly** one solution, and you may not use the *same* element twice.

**Example:**

```
Given nums = [2, 7, 11, 15], target = 9,

Because nums[0] + nums[1] = 2 + 7 = 9,
return [0, 1].
```

**Solution**ï¼š

```c++
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        vector<int> n;
        unordered_map<int, int> m = {};
        for(int i = 0; i < nums.size(); i++) {
            if (m.find(target - nums[i]) != m.end()) {
                n.push_back(i);
                n.push_back(m[target - nums[i]]);
                return n;
            }
            m.emplace(nums[i], i);
        }
        return n;
    }
};
```

The brute force way is using two loops. Apparently, this way does not need more space, so it's a good way to save memory.

But using **Hash Table** can improve the speed, just like this, just using one loop.And using **".emplace" **instead of ".insert" can also improve speed.But because we have to save Hash Table, we waste much space.
